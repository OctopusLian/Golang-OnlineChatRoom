## 设计思路  
使用proto2协议  

通过Protobuf中内置好的编码函数，将要发送的数据进行编码，之后在“编码后的”数据前加入协议号和报头，再转码通过Write与Read函数进行数据的发送和接收。  
#### 版本更新日志  
- 2018.8.28下午北京时间四点：完成服务端与客户端两端收发协议的逻辑，客户端可以成功连接上服务端，并自动发送由protobuf编码后的数据给服务端，服务端不断接收；  
- 2018.8.28下午北京时间五点十分：完成客户端发消息编码后被服务端接收并解码后转发，但存在多客户端连接后没有提示，不能多人聊天的问题；  
- 2018.8.28下午北京时间六点半：解决协议上的字节数据拼接问题；  
- 2018.8.28晚上北京时间七点：去掉登入登出房间的逻辑，服务器端只做数据转发，客户端只要连接上服务端即可开始群聊，用protobuf协议完成了第四版本聊天室的练习；
